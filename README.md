Supermarket Sales Analysis
Project Overview
This project performs an Exploratory Data Analysis (EDA) on a supermarket sales dataset to uncover key insights into customer behavior and sales performance. The goal is to provide actionable recommendations for optimizing sales strategies and improving customer experience. 
GitHub
GitHub

Key Objectives
Identify the highest-performing branches and cities.
Analyze customer demographics (gender and type) to understand purchasing patterns.
Determine the most profitable product lines and popular payment methods.
Visualize sales trends over time to identify peak shopping hours and days. 


Technologies Used
Language: Python
Environment: Jupyter Notebook
Libraries:
Pandas  for data cleaning and manipulation.
Matplotlib for data visualization. 

Dataset Information
The dataset used in this project is sourced from Kaggle. It contains 1,000 historical sales records from three different supermarket branches over a three-month period. 
Attributes: Invoice ID, Branch, City, Customer Type, Gender, Product Line, Unit Price, Quantity, Total, Date, Time, Payment, etc. 

Analysis Steps
Data Cleaning: Handling missing values, checking for duplicates, and ensuring correct data types.
Descriptive Statistics: Summarizing the dataset to understand the distribution of key variables.
Exploratory Data Analysis (EDA):
Univariate Analysis: Examining individual columns like 'Rating' or 'Total'.
Bivariate/Multivariate Analysis: Exploring relationships between variables, such as sales by branch or product line by gender.
